# Project Structure Documentation

## Overview
This repository contains a structured environment for managing agents, assets, and metadata. Below is a detailed breakdown of the directory structure, including the roles and interactions of key components.

## Directory Structure

### `.agents/`
- **Purpose**: This directory is dedicated to managing agent-related metadata and outputs.
- **Key File**: `agent_assets_metadata.toml`
  - **Description**: Contains metadata about uploads and generated outputs.
  - **Structure**:
    ```toml
    uploads = []
    generated = []

    [[outputs]]
    id = "sE-MQQnBwDPTfHCTZ8QWn"
    uri = "file://attached_assets/Evaluation_Ulysse_AI_Mars2026_MiseAJour.pdf"
    type = "pdf"
    title = "Evaluation Ulysse AI - Mars 2026 (Mise a jour 19 mars) - CORRIGE"

    [[outputs]]
    id = "ppjeqwQB9fu1sQf9WqHCH"
    uri = "file://generated_reports/Analyse_SUGU_Mars2026.pdf"
    type = "pdf"
    title = "Analyse Stratégique SUGU — Mars 2026"

    [[outputs]]
    id = "B3_kLvdO7pAYr15DrJl0q"
    uri = "file://generated_reports/Analyse_DevMax_Mars2026.pdf"
    type = "pdf"
    title = "Analyse Stratégique DevMax — Mars 2026"
    ```
  - **Usage**: Metadata is used to track agent-related outputs and their associated files.

### `.canvas/`
- **Purpose**: This directory stores visual assets and resources used in the project.
- **Key Files**:
  - **Image Assets**: Files such as `asset_-1571932399.png`, `asset_-1938668156.png`, etc.
    - **Description**: These are PNG images used for visual representation or documentation purposes.
    - **Significance**: These assets may be used in reports, presentations, or as part of the project's UI.

### Metadata File Formats
- **TOML**: Used for structured metadata storage in `agent_assets_metadata.toml`. This format is human-readable and supports hierarchical data.
- **PNG**: Image files are stored in `.canvas/assets/` for visual purposes.

## Relationships and Usage
- **Agents and Outputs**: The `.agents/` directory tracks outputs generated by agents, linking them to metadata for easy retrieval.
- **Canvas Assets**: Visual assets in `.canvas/` may complement agent outputs or serve standalone purposes in documentation.

## Example Workflow
1. **Agent Metadata**: An agent generates a report, and its metadata is stored in `agent_assets_metadata.toml`.
2. **Visual Representation**: Relevant images from `.canvas/assets/` are used to enhance the report.
3. **Output Delivery**: The final report, along with its metadata and visual assets, is delivered to stakeholders.
