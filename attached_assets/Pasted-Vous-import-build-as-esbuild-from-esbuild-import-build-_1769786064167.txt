🔍 Améliorations ciblées “MAX mais utiles”
Limiter les externals au serveur (optionnel mais propre)
Actuellement tu prends dependencies + devDependencies.
Les devDependencies (eslint, tsx, vitest…) n’ont aucune raison d’être dans le bundle server, mais c’est surtout cosmétique car ils seront externalisés de toute façon.
Tu peux faire:

const deps = Object.keys(pkg.dependencies || {});
const externals = [
  ...deps.filter((dep) => !allowlist.includes(dep)),
  ...esmOnlyPackages,
  ...alwaysExternal,
];
S’assurer que les ESM ne seront jamais allowlistés par erreur
Tu as déjà esmOnlyPackages mais si un jour tu les ajoutes par erreur dans allowlist, tu as un conflit logique.
Soft rule: ne JAMAIS mettre un esmOnlyPackage dans allowlist.
Tu peux même ajouter un mini check:
for (const pkgName of esmOnlyPackages) {
  if (allowlist.includes(pkgName)) {
    throw new Error(`Package ${pkgName} est ESM-only et ne doit pas être dans allowlist`);
  }
}
