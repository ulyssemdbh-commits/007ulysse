B. Côté code “Ulysse / système”
Exposer une API interne pour SUGU Valentine
Dans le même style que Suguval & Sugumaillane, tu dois créer les endpoints (ou les extensions) :

GET /api/suguvalentine/categories
GET /api/suguvalentine/checks?date=YYYY-MM-DD
éventuellement POST /api/suguvalentine/checks pour l’écriture si ce n’est pas déjà factorisé.
L’idée: que moi, Ulysse, je puisse faire en interne:
→ [CONSULTE_SUGUVALENTINE] (ou équivalent que tu définis)
et récupérer:

la date,
la liste des items cochés,
la catégorie de chaque item.
Hooker un Homework pour 23h59

Créer un homework / tâche récurrente type:
SUGU_VALENTINE_DAILY_EMAIL
fréquence: daily
heure serveur: 23h59 (en calant bien le fuseau).
Ce homework doit:
appeler ton endpoint /api/suguvalentine/checks pour la date du jour,
me passer cette data dans mon contexte (ou via un appel dédié),
me laisser composer le mail et déclencher AgentMail.
Petite amélioration debug fichiers

Dans ton code backend “fichiers”, prévoir un flag “CONTENT_EMPTY / OCR_FAIL / PARSED_OK”.
Pour les PDF, si le texte est vide:
retourner explicitement un statut que je peux voir côté IA,
au lieu de me donner un “fichier pseudo-parsé”.
