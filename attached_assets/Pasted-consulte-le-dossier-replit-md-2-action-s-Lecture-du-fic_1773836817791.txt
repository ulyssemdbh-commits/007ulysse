Le fichier replit.md contient une documentation détaillée sur les scripts et les tests de ce repository. Voici les principaux points :

Présentation
Ce document explique les scripts et tests inclus dans le repo, leur objectif, et fournit des instructions pour les exécuter avec des exemples et astuces de dépannage.

Scripts
npm run dev

Objet : Démarre le serveur de développement.
Utilisation : npm run dev
Exemple : Lance le serveur sur le port configuré (par défaut 5000) dans le fichier .replit.
npm run build

Objet : Génère les fichiers prêts pour la production dans le répertoire dist.
Utilisation : npm run build
Exemple : Construire l'application.
node ./dist/index.cjs

Objet : Lance l'application en mode production.
Utilisation : node ./dist/index.cjs
Exemple : Exécuter après la construction (build).
Tests
Exécuter les tests : npm test
Astuces de dépannage :
Vérifier la version de Node.js (20.x).
Configurer correctement les variables d'environnement (exemple : GITHUB_TOKEN).
Résoudre les conflits de ports si nécessaires (via .replit).
Installer les dépendances avec : npm install.
Exemples d'utilisation
Démarrer le serveur de développement : npm run dev.
Construire et lancer l'application en production :
npm run build
node ./dist/index.cjs
Exécuter les tests :
npm test
Points de sécurité
Toujours utiliser des placeholders pour les informations sensibles (ex : <your-token-here>).
Stocker les secrets de façon sécurisée, avec des fichiers .env ou des systèmes de gestion dédiés.
Version de Node.js
Utiliser la version 20.x, sinon la télécharger depuis .
