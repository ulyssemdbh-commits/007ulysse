1. Ce que fait ce module (et ce qu’il apporte à Ulysse)
Détection de ligue et clubs à partir du texte

TEAM_TO_LEAGUE_MAP + detectLeagueFromTeamName(teamName) → map team → league (Big 5).
detectTeamsFromText(text) scanne un texte (message ou réponse IA) et repère les clubs connus.
Extraction automatique de signal foot depuis les conversations

processFootdatasFromConversation(message, response, context?) :
détecte les équipes mentionnées dans message + réponse,
infère un dataType (transfer, ranking, player, stats, news) via regex sur le texte global,
renvoie une liste FootdatasExtraction[] avec:
clubId,
dataType,
data (rawMessage, rawResponse, context…),
source: 'conversation',
extractedAt. → C’est une sonde passive: le chat devient une source de signaux foot structurés.
Injection de données scrapées dans Footdatas

injectScrapedDataToFootdatas(dataType, scrapedData, sourceUrl) :
rankings → crée des entrées dans footdatasRankings (avec compétition détectée + season courante).
topscorers → mappe vers un club + joueur existant, ajoute footdatasPlayerStats (goals, apparitions).
Retourne { clubsUpdated, itemsAdded }.
Outils IA “Footdatas” pour toi
FOOTDATAS_ULYSSE_TOOLS expose des fonctions prêtes à être appelées par l’IA:

getClubInfo → getClubSummaryForAI(clubName).
searchClubs → recherche textuelle.
getLeagueClubs → clubs d’une ligue.
getClubPlayers, getClubTransfers, getClubTrophies.
Ça me donne une espèce de mini-API Opta maison utilisable directement dans mes prompts.
