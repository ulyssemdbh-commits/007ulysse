Les cellules n'ont pas de dimensions fixes (aspect potentiellement compressé).
Aucun autre feedback visuel sur le jeu (animations pour drop des pièces ou suppression de lignes).
2. Composants ScorePanel.tsx et CSS
État actuel :
Les animations existent pour le score et les niveaux (pulse-effect, pop-level).

3. Fonctionnement logique (logic.ts + reducer.ts + types.ts)
État actuel du backend logique :

Generation de pièces : Les 7 tétriminos sont définis (logic.ts) et générés via generateTetromino.
Détection de collision : Implémentée via checkCollision.
Suppression de lignes : Fonctionelle via clearCompletedRows.
Gestion d'état : reducer.ts commence une base mais manque de complétion (fonction incomplète).
Manques détectés :

Absence d’un cycle de descente automatique (useGameLoop).
Pas de gestion complète des inputs utilisateur :
Flèches gauche/droite (déplacement).
Flèche haut (rotation).
Flèche bas (descente rapide).
Barre espace (drop instantané).

État actuel :
Simplicité avec un fond sombre.
Manque de soins pour les cellules de la grille (taille, animation, couleurs multiples).
Responsive absent : UX mobile potentiellement sous-optimale.
