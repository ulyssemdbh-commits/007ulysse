2.2. Pipeline interne du service TTS
Pour chaque requête:

Normalisation du texte:

trim, suppression double espaces
découpage en phrases (mêmes règles que ton pipeline v4.1 côté LLM)
option: regex pour nettoyer trucs inutiles (“…” exagérés, etc.)
Cache audio (fort gain réel):

key = hash(voice + text_normalisé)
stockage:
soit en RAM (LRU cache),
soit sur disque (/var/cache/piper) avec index.
si hit → renvoi immédiat (10–50ms), aucun appel Piper.
Planning:

file interne:
priorité haute pour phrases courtes et messages système,
priorité normale pour tout le reste.
limite de requêtes parallèles pour éviter d’exploser la machine (genre 2–4 synthèses concurrentes max).
Synthèse avec Piper:

soit via bindings Python,
soit via CLI:
echo "texte" | piper --model fr_FR-... --output_file -
résultat: buffer WAV → transcodé vers MP3/OGG (via ffmpeg ou lib interne).
Streaming:

si utilisé avec ton pipeline TTS phrase‑par‑phrase:
tu peux renvoyer l’audio phrase par phrase,
ou tout concaténer si tu veux un call POST /tts global.

3.1. Nouveau ttsOneSentence côté Node (backend Ulysse)
const TTS_ENDPOINT = process.env.TTS_ENDPOINT || "http://tts-service:8080/tts";

async function ttsOneSentence(text: string, voice: string): Promise<string | null> {
  const trimmed = text.trim();
  if (trimmed.length < 2) return null;

  try {
    const resp = await fetch(TTS_ENDPOINT, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        text: trimmed,
        voice,          // "fr_fast", "fr_natural", etc.
        format: "mp3",
        cache: true,
        speed: 1.0
      }),
    });

    if (!resp.ok) {
      console.error("[TTS] HTTP error", resp.status);
      return null;
    }

    // Mode base64 (compatible avec audio_chunk actuel)
    const json = await resp.json();
    if (!json.audio_base64) {
      console.error("[TTS] Missing audio_base64");
      return null;
    }

    return json.audio_base64 as string;
  } catch (err) {
    console.error("[TTS] Error:", err);
    return null;
  }
}
Ton pipeline v4.1 reste identique: TTS en parallèle, lecture séquentielle, barge‑in, etc.
Tu as juste changé le fournisseur TTS.
