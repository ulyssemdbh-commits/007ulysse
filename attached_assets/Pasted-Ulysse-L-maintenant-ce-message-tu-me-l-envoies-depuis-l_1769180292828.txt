2️⃣ Ce que ton backend attend vraiment
Pour que ça marche, il faut cette séquence :

Connexion WebSocket à /ws/voice OK.
Le client envoie un JSON :
{ "type": "auth", "userName": "Maurice" }
→ le serveur valide le cookie et répond :
{"type":"authenticated", ...}
Ensuite seulement, le client doit envoyer :
{"type":"start_listening", "conversationId": ...}
puis envoyer les chunks audio binaires (Buffers)
puis {"type":"stop_listening"} pour déclencher processAudioAndRespond.
Si l’un de ces trucs n’est pas envoyé, le serveur ne fait rien, même si tu parles.
