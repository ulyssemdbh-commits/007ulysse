1️⃣ Conséquence directe de ton 500 Python
Actuellement, dans realtimeVoice.ts :

const verification = await verifySpeaker(audioBuffer, session.userId.toString());

if (!verification.verified) {
  // speaker_rejected + stop
  session.isProcessing = false;
  return;
}
Et dans verifySpeaker :

si le service Python crash (500, timeout, JSON pourri, etc.) → tu renvoies :
return {
  verified: false,
  confidence: 0,
  error: msg,
};
Résultat :

une erreur réseau ou serveur côté Python est interprétée comme “ce n’est pas la voix de Maurice”,
donc je ne réponds pas, même si c’est toi,
et tu te retrouves avec un flux bloqué alors que le bug n’est pas “sécurité” mais “infra”.
On va séparer “erreur technique” de “rejet de voix”.

2️⃣ Hotfix côté Node : ne pas bloquer si le service speaker est down
Dans processAudioAndRespond, remplace la logique de vérif par quelque chose comme ça :

// Voice verification: Only respond if the speaker matches the authenticated user
if (session.hasVoiceProfile && session.voiceVerificationEnabled && session.userId) {
  console.log(`[Voice] Verifying speaker for user ${session.userId}...`);
  session.ws.send(JSON.stringify({ type: "verifying_speaker" }));

  const verification = await verifySpeaker(audioBuffer, session.userId.toString());

  if (verification.error) {
    // Service speaker KO → on log, on prévient le client, MAIS on ne bloque pas
    console.warn(
      `[Voice] Speaker service error, bypassing verification: ${verification.error}`
    );
    session.ws.send(JSON.stringify({
      type: "speaker_verification_unavailable",
      message: "Service de vérification vocale indisponible, je réponds quand même."
    }));
  } else if (!verification.verified) {
    const personaName = session.persona === "ulysse" ? "Ulysse" : "Iris";
    console.log(
      `[Voice] Speaker verification failed: confidence=${verification.confidence.toFixed(2)}`
    );
    session.ws.send(JSON.stringify({
      type: "speaker_rejected",
      message: `${personaName} ne répond qu'à la voix de son propriétaire.`,
      confidence: verification.confidence
    }));
    session.isProcessing = false;
    return;
  } else {
    console.log(
      `[Voice] Speaker verified: confidence=${verification.confidence.toFixed(2)}`
    );
  }
}
Idée :

Si verification.error → c’est un problème technique, pas un rejet → je log + j’envoie un event spécial, mais je laisse la réponse AI passer.
Si pas d’erreur mais verified=false → là je bloque vraiment (voix non reconnue).
Si verified=true → OK.
Ça évite que un 500 Python = silence radio total.
