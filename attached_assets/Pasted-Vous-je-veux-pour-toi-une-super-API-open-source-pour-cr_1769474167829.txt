1️⃣ Le vrai top aujourd’hui
Pour du crawler + scraper sérieux, scalable, tous usages, le meilleur écosystème open source actuel c’est :

🧠 Crawlee (ex-Apify SDK)
Langages : TypeScript / JavaScript
Points forts :
Gère crawler + scraper proprement (HTTP, headless browser, proxy, anti-ban, file d’attente d’URLs…)
Support Cheerio (HTML server-side) + Playwright / Puppeteer (sites JS lourds)
Gestion du rate limiting, retries, sessions, cookies
Stockage intégré (filesystem, S3, etc.), middlewares, hooks
C’est pensé pour faire du “tous sujets / tous cas”, exactement ce qu’il te faut pour que je puisse bosser sur tout.
👉 Autrement dit : tu me mets Crawlee côté serveur, je peux:

crawler un site de foot, un e-commerce, des docs, des classements,

2️⃣ Stack que je te recommande pour “Ulysse”
Vu que tu es déjà en Node/TS, je ferais ça :

Core crawling / scraping : crawlee
Parsing HTML simple : cheerio
Browser headless pour les sites JS : crawlee + playwright
Pipeline :
un service crawlerService générique

Exemple de pattern d’API que tu exposes pour moi (côté backend) :

// POST /api/crawl
// body: { url: string, mode?: "auto" | "http" | "browser" }
Je t’envoie une URL + éventuellement le “mode”

Me renvoie un JSON normalisé : { html, text, links, metadata }
Et derrière, j’analyse/structure en fonction du besoin (pronos, actus, etc.).
