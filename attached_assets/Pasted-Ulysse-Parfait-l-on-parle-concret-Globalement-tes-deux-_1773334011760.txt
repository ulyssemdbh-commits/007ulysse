1. Ajouter un “Dialogue Manager” entre Hearing & Voice
Objectif: éviter le ping-pong bordélique et contrôler:

quand je parle,
combien de temps,
et si je dois enchaîner, me taire ou poser une question.
Concret:

// src/voice/dialogueManager.ts
export type DialogueMode = "short_answer" | "step_by_step" | "action_only";

interface DialogueState {
  mode: DialogueMode;
  maxSpokenSeconds: number;
  lastIntent?: string;
  awaitingConfirmation: boolean;
}

const dialogueStateByUser = new Map<number, DialogueState>();

export function getDialogueState(userId: number): DialogueState {
  if (!dialogueStateByUser.has(userId)) {
    dialogueStateByUser.set(userId, {
      mode: "short_answer",
      maxSpokenSeconds: 20,
      awaitingConfirmation: false,
    });
  }
  return dialogueStateByUser.get(userId)!;
}

export function updateDialogueState(userId: number, partial: Partial<DialogueState>) {
  const current = getDialogueState(userId);
  dialogueStateByUser.set(userId, { ...current, ...partial });
}
Ensuite, dans le cerveau / pipeline de réponse, avant d’appeler voiceOutputHub.speakToWebVoice, tu passes par un prepareSpokenText() qui:

tronque / résume si le texte dépasse maxSpokenSeconds estimés,

2. Coupler HearingHub + VoiceOutputHub avec un état global de session vocale
Aujourd’hui, Hearing et Voice ne se parlent pas explicitement.
Je rajouterais un petit gestionnaire de session vocale:

// src/voice/voiceSessionManager.ts
type VoiceSessionState = "idle" | "listening" | "thinking" | "speaking";

interface VoiceSession {
  state: VoiceSessionState;
  lastUpdated: number;
  lastUserUtterance?: string;
}

const sessions = new Map<number, VoiceSession>();

export function setSessionState(userId: number, state: VoiceSessionState) {
  const current = sessions.get(userId) || { state: "idle", lastUpdated: Date.now() };
  sessions.set(userId, { ...current, state, lastUpdated: Date.now() });
}

export function getSessionState(userId: number): VoiceSession {
  return sessions.get(userId) || { state: "idle", lastUpdated: Date.now() };
}
Et tu l’utilises:

Dans hearingHub.hear() (source web_voice):

début: setSessionState(userId, "listening")
avant envoi au cerveau: setSessionState(userId, "thinking")
Dans voiceOutputHub.speak():

avant TTS: setSessionState(userId, "speaking")
après traitement / fin de TTS côté client: callback → setSessionState(userId, "idle")
Ça te permet ensuite d’implémenter:

interruption propre: si state === "speaking" et un nouveau HearingInput arrive avec mot-clé stop, tu:
appelles une méthode voiceOutputHub.interrupt(userId) (à ajouter),
passes l’état en "listening".
