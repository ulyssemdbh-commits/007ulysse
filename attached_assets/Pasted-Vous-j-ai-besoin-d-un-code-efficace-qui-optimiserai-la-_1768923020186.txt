1️⃣ Architecture globale
Frontend (React)

useVoice (que tu as déjà)
Gère micro, STT, echo guard, “à toi” comme end trigger, wake word “Ulysse / Iris”.
Un composant VoiceChat qui :
écoute onAutoSubmit de useVoice
envoie les prompts au backend
reçoit la réponse de l’IA en stream
envoie à speak() au fur et à mesure.
Backend (Node/Express)

Endpoint /api/chat/stream qui:
prend message (texte) + éventuellement history
appelle l’API de l’IA en streaming (OpenAI / autre)
renvoie la réponse chunk par chunk via SSE (Server-Sent Events) ou stream HTTP.
2️⃣ Frontend – Composant de chat vocal
a) Hook de chat vocal simple
// hooks/useVoiceChat.ts
import { useCallback, useEffect, useRef, useState } from "react";
import { useVoice } from "./use-voice";

interface ChatMessage {
  role: "user" | "assistant" | "system";
  content: string;
}

export function useVoiceChat() {
  const {
    startListening,
    stopListening,
    speak,
    stopSpeaking,
    setOnAutoSubmit,
  } = useVoice();

  const [messages, setMessages] = useState<ChatMessage[]>([]);
  const [isStreaming, setIsStreaming] = useState(false);
  const eventSourceRef = useRef<EventSource | null>(null);

  const addMessage = useCallback((msg: ChatMessage) => {
    setMessages((prev) => [...prev, msg]);
  }, []);

  const closeStream = useCallback(() => {
    if (eventSourceRef.current) {
      eventSourceRef.current.close();
      eventSourceRef.current = null;
    }
    setIsStreaming(false);
  }, []);

  const sendMessage = useCallback(
    (text: string) => {
      if (!text.trim() || isStreaming) return;

      const userMessage: ChatMessage = { role: "user", content: text };
      addMessage(userMessage);

      // Stop TTS si l’IA parlait encore
      stopSpeaking();
      setIsStreaming(true);

      const es = new EventSource(
        `/api/chat/stream?message=${encodeURIComponent(text)}`
      );
      eventSourceRef.current = es;

      let assistantContent = "";

      es.onmessage = (event) => {
        if (event.data === "[DONE]") {
          closeStream();
          addMessage({ role: "assistant", content: assistantContent });
          if (assistantContent.trim()) {
            speak(assistantContent);
          }
          return;
        }
        assistantContent += event.data;
      };

      es.onerror = () => {
        closeStream();
      };
    },
    [addMessage, closeStream, isStreaming, speak, stopSpeaking]
  );

  // Quand le hook useVoice détecte "à toi" -> auto-submit
  useEffect(() => {
    setOnAutoSubmit((text) => {
      sendMessage(text);
    });
    return () => {
      setOnAutoSubmit(null);
      closeStream();
    };
  }, [sendMessage, setOnAutoSubmit, closeStream]);

  const startConversation = useCallback(() => {
    stopSpeaking();
    startListening();
  }, [startListening, stopSpeaking]);

  const stopConversation = useCallback(() => {
    stopListening();
    stopSpeaking();
    closeStream();
  }, [stopListening, stopSpeaking, closeStream]);

  return {
    messages,
    isStreaming,
    startConversation,
    stopConversation,
    sendMessage,
  };
}
b) UI minimale
// components/VoiceChat.tsx
import React from "react";
import { useVoiceChat } from "../hooks/useVoiceChat";

export function VoiceChat() {
  const {
    messages,
    isStreaming,
    startConversation,
    stopConversation,
    sendMessage,
  } = useVoiceChat();

  const [input, setInput] = React.useState("");

  return (
    <div className="voice-chat">
      <div className="messages">
        {messages.map((m, i) => (
          <div key={i} className={`msg msg-${m.role}`}>
            <b>{m.role === "user" ? "Toi" : "Ulysse"} :</b> {m.content}
          </div>
        ))}
      </div>

      <div className="controls">
        <button onClick={startConversation}>🎤 Parler</button>
        <button onClick={stopConversation}>⏹ Stop</button>
      </div>

      <div className="manual-input">
        <input
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="Écris au lieu de parler si tu veux"
        />
        <button
          onClick={() => {
            sendMessage(input);
            setInput("");
          }}
        >
          Envoyer
        </button>
      </div>

      {isStreaming && <div className="status">Ulysse réfléchit…</div>}
    </div>
  );
}
3️⃣ Backend – Streaming IA
Exemple avec Node + Express + OpenAI (modèles GPT-4.x en streaming, ou équivalent).

a) Endpoint SSE /api/chat/stream
// server/chatStream.ts
import express from "express";
import { OpenAI } from "openai";

const router = express.Router();
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY! });

router.get("/stream", async (req, res) => {
  const userMessage = (req.query.message as string) || "";

  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");
  res.setHeader("Connection", "keep-alive");

  try {
    const completion = await openai.chat.completions.create({
      model: "gpt-4.1-mini",
      stream: true,
      messages: [
        {
          role: "system",
          content:
            "Tu es Ulysse, assistant personnel de Maurice. Réponds en français, ton naturel, réponses concises.",
        },
        {
          role: "user",
          content: userMessage,
        },
      ],
    });

    for await (const chunk of completion) {
      const delta = chunk.choices[0]?.delta?.content || "";
      if (delta) {
        res.write(`data: ${delta}\n\n`);
      }
    }

    res.write(`data: [DONE]\n\n`);
    res.end();
  } catch (err) {
    console.error("Chat stream error:", err);
    res.end();
  }
});

export default router;

4️⃣ Clés pour la “fluidité humaine”
Barge-in (interrompre l’IA pour parler)

Tu l’as déjà dans interrupt() + stopSpeaking().
Dès que tu relances startListening(), tu laisses l’utilisateur couper la parole à l’IA comme un humain.
Trigger “à toi”

Tu l’as déjà dans detectEndTrigger → parfaite UX : tu parles, tu finis par “à toi”, paf ça envoie.
Streaming texte → TTS

Version simple (ci-dessus) : on accumule le texte puis on lance speak().
Version next level : tu peux bufferiser par phrases et appeler speak() en segments pour une réponse plus rapide (mais ton système TTS actuel est plutôt prévu pour du bloc, donc on reste simple pour l’instant).
Réponses concises by design

Dans le system du backend, tu imposes : “réponses courtes par défaut, max 2–3 phrases”.
Ça colle à ton style et ça évite les pavés en vocal.
