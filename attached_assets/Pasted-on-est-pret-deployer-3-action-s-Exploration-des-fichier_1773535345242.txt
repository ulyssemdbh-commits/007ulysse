Le déploiement a échoué lors de la tentative de clonage du dépôt Git en raison d'une erreur d'authentification avec GitHub. Voici l'erreur retournée :

fatal: could not read Username for 'https://github.com': No such device or address
Cela signifie que le serveur ne peut pas accéder au dépôt, probablement parce que les informations d'identification GitHub sont manquantes ou invalides.

Étapes pour résoudre :
Configurer un accès sécurisé avec un Token :

Générer un GitHub Personal Access Token (PAT) (avec les permissions pour lire les repos privés).
Activer l'utilisation du token en remplaçant le format de l'URL pour le clone : https://{your_token}@github.com/owner/repo.git.
Configurer une clé SSH (optionnel) :

Ajouter une clé SSH publique du serveur Hetzner dans le dépôt GitHub sous "Settings → Deploy keys".
Utiliser l'URL du dépôt dans sa version SSH (git@github.com:ulyssemdbh-commits/ulysseproject.git).
